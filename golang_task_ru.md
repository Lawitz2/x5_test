## Цель
Реализовать минимальную систему, которая принимает заказы, сохраняет их и выполняет обработку асинхронно.

---

## Функциональные требования

### Доменная модель (можно менять)
**Order (Заказ)**
- `id` (UUID или numeric)
- `customer_id` (string/int)
- `items` (любое представление)
- `status` (например: `NEW`, `PROCESSING`, `FULFILLED`, `FAILED`)
- `created_at`, `updated_at`

---

## HTTP API (Echo)

### 1) Создать заказ
`POST /orders`

Пример тела запроса:
```json
{
  "customer_id": "cust-123",
  "items": [
    {"sku": "ABC", "qty": 2},
    {"sku": "XYZ", "qty": 1}
  ]
}
```

Ответ:
- `201 Created` с созданным заказом (или хотя бы `id` и `status`)

Поведение:
- Заказ должен быть сохранён в PostgreSQL.
- После сохранения должна запускаться асинхронная обработка заказа.

### 2) Получить заказ
`GET /orders/:id`

Возвращает заказ с текущим статусом.

### 3) Список заказов (минимально)
`GET /orders?customer_id=...&status=...`

---

## Обработка (gRPC)
Обработка заказа должна вызываться через **gRPC**.

Поведение:
- установить статус `PROCESSING`
- выполнить симуляцию операции (например, sleep 100–500ms, валидация items, случайная ошибка и т.п.)
- обновить статус на `FULFILLED` или `FAILED`

---

## PostgreSQL (pgx)
- Используйте PostgreSQL для хранения заказов.
- Используйте **pgx** для доступа к БД.
- Добавьте SQL-миграции или init-скрипт для схемы.

## Требования к Docker Compose
`docker-compose.yml` должен поднимать:
- PostgreSQL

Должна быть возможность выполнить:

После запуска должно быть возможно:
- вызывать HTTP-эндпоинты
