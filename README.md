# Orders & Fulfillment Service

Это минимальная система для приема заказов, их сохранения в PostgreSQL и асинхронной обработки через gRPC.

## Технологический стек
- Golang
- PostgreSQL
- pgx
- Echo (HTTP API)
- gRPC (для внутренней коммуникации)

## Запуск системы

### 1. Сборка и запуск с помощью Docker Compose

Убедитесь, что у вас установлены Docker и Docker Compose.

1.  **Сборка и запуск:**
    ```bash
    docker compose up --build
    ```
    Эта команда соберет Docker-образ приложения и запустит PostgreSQL, а затем само приложение.

2.  **Проверка состояния:**
    Убедитесь, что оба сервиса запущены и приложение подключилось к БД:
    ```bash
    docker compose ps
    ```
    Вы должны увидеть `x5_test_db` в состоянии `healthy` и `x5_test_app` в состоянии `Up`.

### 2. Использование HTTP API (Echo)

Приложение доступно по портам, указанным в `.env.example` (по умолчанию HTTP на `8080`, gRPC на `50051`).

#### 2.1. Создание заказа (`POST /orders`)

Создает новый заказ.

**Запрос:**
```bash
curl.exe -X POST http://localhost:8080/orders -H "Content-Type: application/json" -d '{
  "customer_id": "cust-1",
  "items": [
    {"sku": "SKU-A100", "qty": 2},
    {"sku": "SKU-B200", "qty": 5}
  ]
}'
```


**Ожидаемый ответ (201 Created):**
```json
{
  "id": "a1b2c3d4-e5f6-7890-1234-567890abcdef",
  "customer_id": "customer-alpha-1",
  "items": [
    {"sku": "SKU-A100", "qty": 2},
    {"sku": "SKU-B200", "qty": 5}
  ],
  "status": "NEW",
  "created_at": "2024-01-01T10:00:00Z",
  "updated_at": "2024-01-01T10:00:00Z"
}
```
*(ID и время будут сгенерированы сервером)*

#### 2.2. Получение заказа (`GET /orders/:id`)

Получает заказ по его ID.

**Запрос (используя ID из предыдущего шага):**
```bash
curl -X GET http://localhost:8080/orders/a1b2c3d4-e5f6-7890-1234-567890abcdef
```

**Ожидаемый ответ (200 OK):**
Статус будет меняться в зависимости от того, успела ли обработка завершиться (`NEW` -> `PROCESSING` -> `FULFILLED` или `FAILED`).

#### 2.3. Список заказов (`GET /orders`)

Получает список заказов с возможностью фильтрации.

**Запрос (все заказы):**
```bash
curl -X GET "http://localhost:8080/orders"
```

**Запрос (фильтр по клиенту):**
```bash
curl -X GET "http://localhost:8080/orders?customer_id=cust-1"
```

**Запрос (фильтр по статусу):**
```bash
curl -X GET "http://localhost:8080/orders?status=FULFILLED"
```

## Примечания к реализации

1.  **Асинхронная обработка:** Обработка заказа запускается в фоновой горутине после успешного сохранения в БД.
2.  **Воркер (Polling):** В приложении реализован простой воркер (`orderProcessor`), который каждые 5 секунд опрашивает базу данных на наличие заказов со статусом `NEW` и инициирует их gRPC обработку. (Опциональный пункт)
3.  **gRPC Сервер:** Сервер прослушивает порт `50051` и реализует метод `ProcessOrder`.
4.  **Конфигурация:** Используется файл `.env.example` для настройки портов и подключения к БД. В `docker-compose.yml` эти значения переопределены для работы с контейнером БД.
